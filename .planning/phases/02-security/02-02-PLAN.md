<objective>
Refactor backend/server.py to replace the global audio_loop singleton with a per-session dict keyed by Socket.IO sid. Each client gets an isolated AudioLoop; disconnecting a client tears down only its loop. This eliminates a multi-client race condition and a memory leak on disconnect (SEC-04, SEC-05).

Purpose: The global singleton means a second client connection silently hijacks the first client's session. Proper per-sid isolation is required for production safety.
Output: backend/server.py with audio_loops dict, loop_tasks dict, updated event handlers, updated signal handler, and refactored monitor_printers_loop.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| socket.io client→server | Each sid is an untrusted string from the network; used as dict key, never eval'd |
| AudioLoop per-sid | Each session's loop is isolated; one client cannot access another's loop |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Information Disclosure | audio_loops dict | mitigate | All access via audio_loops.get(sid) — a client can only reach its own loop; no cross-sid lookup possible |
| T-02-05 | Denial of Service | disconnect handler | mitigate | loop.stop() and task.cancel() on disconnect prevent zombie tasks accumulating memory/CPU |
| T-02-06 | Tampering | monitor_printers_loop loop_ref | mitigate | loop_ref captured at task creation time — cannot be swapped out by later events |
</threat_model>
