<objective>
Harden Electron security by enabling contextIsolation and removing direct Node.js access from the renderer process. A new preload.js exposes a minimal, typed API surface via contextBridge that replaces all direct ipcRenderer and shell usage.

Purpose: contextIsolation: false is a known critical vulnerability — any XSS in the renderer grants full Node.js access to the attacker. This plan closes that surface (SEC-01, SEC-02, SEC-03).
Output: preload.js (new), updated main.js webPreferences, updated App.jsx window controls, updated PrinterWindow.jsx external links.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| renderer→main | ipcRenderer messages cross from untrusted renderer to privileged main process |
| preload→renderer | contextBridge exposes a typed surface; renderer cannot call arbitrary ipc channels |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Elevation of Privilege | renderer process (App.jsx) | mitigate | contextIsolation: true + nodeIntegration: false prevents renderer from requiring Node modules; only the 4 exposed api methods are callable |
| T-02-02 | Tampering | electron/preload.js openExternal | mitigate | shell.openExternal is called only from preload (trusted context); renderer passes URL as string, no eval or require path |
| T-02-03 | Information Disclosure | contextBridge api surface | accept | surface is 4 known safe methods; no secrets or file paths are exposed |
</threat_model>
